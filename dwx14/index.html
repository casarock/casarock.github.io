Let's take a look at Firefox OS and it's architecture.

Architecture

This is a quick overview what the different layers are.

Let's have a detailed look.

  • Gecko lays directly above a linux layer (Gonk)
  • On top of platform are Apps - 3rd Party and System apps.
  • Access to Hardware features is provided by WebAPIs
  • Access to sensible data (Contacts, phone etc.) is provided by Web Activities
  • GAIA is the UI Layer (HTML5). On top of Gaia is every App!

Gonk

Low level OS of Firefox OS. Linux - based on Android Open Source Project

  • lower level operating system of the Firefox OS
  • Linux kernel (based on the Android Open Source Project (AOSP))
  • userspace hardware abstraction layer (HAL)

Gecko

  • includes a networking stack
  • includes graphics stack
  • includes layout engine
  • includes a JavaScript virtual machine

Gaia

UI level of Firefox OS

Only interface to the underlying operating system and hardware

FRAGMENT: Only interface to the underlying operating system and hardware

  • ... through standard Web APIs implemented by Gecko
  • entirely written in HTML, CSS, and JavaScript

Web APIs and Web activities

  • Web APIs
    • Access device hardware
    • Provides access to data storage
    • Security - three layers
    • With standards in mind(W3C)
  • Web Activities
    • App requests data from an other app
    • e.g. Dial a number requests Phone app

Apps and 3rd party apps

Every HTML5, Javascript, CSS based Apps for Firefox OS

Using WebAPIs and Web Activities

Every App on Firefox OS uses Web activities and Web APIs. No matter if system App (dialer, Contacts etc) or 3rd Party Apps. Every App relies on them.

App Development

Open Web Apps

Apps for Firefox OS are HTML5 apps with an extra manifest file
secured by a three layer security model allowing more or less access depending on the location of the app
You can find and install them either from the web or the marketplace

3 different App types

Hosted Apps

WEB basically every html5 webpage, selfhosted.

Privileged Apps

PRIVILEGED Authenticated Apps e.g. by Marketplace

Certified Apps

CERTIFIED authenticated app that is intended for critical system functions (dialer, etc)

Example: Battery API - cont.

APIs are event driven!


var battery = navigator.battery ||
              navigator.mozBattery ||
              navigator.webkitBattery;

function updateBatteryStatus() {
    console.log("Battery status: " + battery.level * 100 + " %");

    if (battery.charging) {
        console.log("Battery is charging");
    }
}

battery.addEventListener("chargingchange", updateBatteryStatus);
battery.addEventListener("levelchange", updateBatteryStatus);
updateBatteryStatus();

How to distribute your App

Hosted App

  • Host the App on your web space
  • Provide installation using WebAPI
FRAGMENT: Host the App on your Web Provide installation using WebAPI

Hosted App distribution

Check if app is already installed


var request = navigator.mozApps.checkInstalled(manifestPath);

request.onerror = function() {
      console.log('Error checking for installed app: ', request.error.name);
};

request.onsuccess = function() {
    // If the app is installed, you'll get a mozApp object in `request.result`,
    // else `request.result` is null
    console.log("Could be installed: ", request.result !== null ? "is installed" : "is not installed");
};

Hosted App distribution

Install your app


var installRequest = navigator.mozApps.install(manifestPath);

installRequest.onsuccess = function() {
    // No error
    console.log("App installed");
};

installRequest.onerror = function() {
    console.log('Error installing the app: ', installRequest.error.name);
};
